# ha-addons-beta — Agent Guide

## This repo is the SOURCE OF TRUTH for the LibreCoach add-on (Model B)

Author **and** test the LibreCoach add-on **here**, in `ha-addons-beta`. The production repo
`../ha-addons` is a **downstream mirror** — never author the add-on there. The shared add-on
source is pushed up to prod for release; prod's `librecoach/` should only ever change via the
mirror.

- **Source of truth:** `ha-addons-beta` (this repo) — all editing and verification.
- **Prod:** `/home/ted/src/librecoach/ha-addons`, branch `fix/c1-c8-startup-hardening`.
- **Release:** merge `fix/c1-c8-startup-hardening` → `main` in `ha-addons` (gated — explicit
  approval only; it triggers the prod build that reaches end users).

Full procedure: **`beta-notes/beta_sync_plan.md`**. Read it before any sync or release work.

## Syncing beta → prod

```bash
beta-notes/mirror.sh            # dry run (shows real content changes, writes nothing)
beta-notes/mirror.sh --apply    # mirror librecoach/ up to prod
```

- Synced surface is **`librecoach/` only, minus `config.yaml`**. Everything outside
  `librecoach/` (repo root, `.github/`, `can-mqtt-bridge/`, `beta-notes/`) is intentionally
  repo-specific and is never synced.
- `config.yaml` is excluded (Option A): prod keeps its own `version:` and
  `image: …-librecoach` (beta uses `…-librecoach-beta`). When you change `options:`/`schema:`
  in beta, **hand-port the same edits to prod** and keep prod's version/image lines. The mirror
  runs a drift check that flags anything beyond version/image.

## Gotchas

- **Node-RED is an upstream build dependency.** The add-on `Dockerfile` clones
  `librecoach-node-red` **main** at build time and asserts `artifact/flows.json` exists. Before
  any beta or prod build, node-red `main` must be current with a freshly rebuilt
  `artifact/flows.json` (generated by the flow-splitter on Deploy inside Node-RED — not the
  wiring-map tool).
- **`rtk` hook mangles `diff`/`sha256sum`/`git diff`** into non-authoritative summaries. For
  real diffs/patches use `rtk proxy git diff`; for byte-exact comparisons use pure Python
  (`hashlib`/`difflib`), not the shell tools.
- **Line endings:** LICENSE / CONTRIBUTING / source files are LF in this repo (canonical).
  The first mirror normalizes prod from CRLF → LF.
